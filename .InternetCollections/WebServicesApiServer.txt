OpenAPI Specification (OAS) is the standard for describing RESTful HTTP APIs 
in a machine-readable way. It was formerly known as the Swagger Specification 
and can be used for generating documentation, client SDKs, MCP servers, 
"collections" for Postman, Bruno, and much more. 

See swagger.io/specification, spec.openapis.org/oas/v3.1.0.html, learn.openapis.org, 

https://swagger.io/specification, 
https://spec.openapis.org/oas/v3.1.0.html, 
https://learn.openapis.org, 

and many resources more. A description of a web services API following the OAS 
is a OpenAPI Description (OAD). This is the basis, the fundament, the starting
 point. The rest is derived from it.
 
The OAD I'm working at was based on the OAD that Alexandre Elisé worked on.
